description: Craft responses to Google, TripAdvisor, or any online review — positive, negative, or aggressive. Use when the operator pastes a review and wants help responding, or asks how to handle a bad review. No MCP needed — pure writing skill.
---

Use $ARGUMENTS as the review text to respond to. If no review is pasted, ask for it.

**Identify the review type first:**
- **Positive** — thank sincerely, mention something specific from the review, invite them back
- **Constructive negative** — acknowledge the issue, apologize without being defensive, explain what changes if any, invite them to return
- **Aggressive/unfair** — stay professional, address facts calmly, don't escalate, protect the brand's dignity without groveling

**Rules for every response:**
- Under 5 sentences — reviewers and future readers won't read more
- Never generic ("Thank you for your feedback!") — always reference something specific from the review
- Never defensive or passive-aggressive — future readers judge the restaurant by the response more than the review
- Sign with the owner/manager name if provided, otherwise use the restaurant name
- For negative reviews: acknowledge → empathize → one concrete action → invite back. In that order.

**For aggressive or dishonest reviews:**
Don't try to win. The audience is future diners reading the exchange. Respond as if speaking to them, not to the reviewer. One calm, factual sentence addressing the claim, then close.

Draft the response in the same language as the review. After presenting it, offer a shorter version if the response is over 3 sentences.
