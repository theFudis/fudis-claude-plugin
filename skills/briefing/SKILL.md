---
description: Daily operations briefing for the restaurant. Use when the operator asks about today's status, performance, reservations, or wants a morning/evening summary.
---

Run a complete daily briefing using these Fudis tools in order:

1. `list_my_restaurants` — confirm which restaurant is active
2. `list_bookings` — today's reservations: confirmed count, seated, VIPs flagged
3. `daily_actions` — priority actions for today (follow-ups, at-risk customers, campaigns)
4. `customer_segments` — quick segment health (champions, at-risk, lost)
5. `overview` — current menu item count and average rating

Lead with reservations and VIP alerts, then priority actions, then segment health. Keep it concise — operators read this at the start of their shift.

Use $ARGUMENTS to accept an optional date (defaults to today).
